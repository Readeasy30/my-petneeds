## Goal

Build simple, fast, useful websites and tools that can grow traffic, earn income, and be managed by one person using ChatGPT, Codex, GitHub, Canva, and Cloudflare Pages.

## Workflow

- ChatGPT manages strategy and priorities.
- Codex performs safe repository work directly.
- GitHub stores files and commit history.
- Cloudflare Pages publishes from GitHub when used.
- Gerry should not be asked to paste or manually edit files unless there is a real blocker.

## Remote workspace rule

- Use Codex remote/cloud workspace for repo edits, tests, file creation, file updates, and commits whenever available.
- Do not require Gerry's local Windows computer, local Git, terminal Git, VS Code, GitHub Desktop, downloads, or manual file copying for routine work.
- If local computer work is unavoidable, give the smallest possible step list and explain why.

## Chrome plugin rule

- Use approved browser plugins/extensions only when they safely help research, page review, screenshots, accessibility checks, SEO checks, grammar review, or workflow assistance.
- Do not require Gerry's local Chrome profile, bookmarks, or extensions for normal repo work.
- Do not rely on unknown or risky extensions.
- If a plugin needs risky access or account setup, stop and report the blocker.

## Mobile access rule

- Support phone/tablet access for review and control.
- Gerry may use mobile to review progress, approve safe work when needed, check GitHub, check Cloudflare Pages, and view deployed sites.
- Keep summaries short and mobile-readable.
- Do not require mobile code editing, mobile Git, terminal work, downloads, or copy/paste file replacement.
- If mobile is useful, give the smallest tap-by-tap steps.

## Codex rules

- Read repo instructions first.
- Keep the current tech stack.
- Make safe updates in batches.
- Commit each safe batch with a clear message.
- Record blockers in a status file.
- Prefer simple, stable, mobile-first HTML/CSS/JS unless the repo already uses another stack.

Do not add secrets, live ads, tracking scripts, payment setup, public upload systems, user accounts, framework rebuilds, or major code deletion without direct approval.

## Operating rule

Do not waste Gerry's time. Move work forward safely. Ask fewer questions. Stop only for true blockers or major production risk.
