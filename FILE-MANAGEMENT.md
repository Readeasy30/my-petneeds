# FILE MANAGEMENT

## Main rule

Update files directly through repository access when possible.

Avoid asking the owner to manually paste large updates.

## Safe update workflow

1. Read existing file first.
2. Preserve working structure.
3. Avoid deleting working content.
4. Prefer clear full-file updates when cleaner.
5. Commit useful safe changes.
6. Update sitemap.xml when adding public pages.
7. Keep navigation and footer links consistent.

## Safe file types

- HTML
- CSS
- JavaScript
- Markdown
- robots.txt
- sitemap.xml

## Avoid

- private keys
- payment systems
- live tracking scripts
- unnecessary frameworks
- fragile experimental rewrites

## Mobile-first rule

Always preserve mobile usability when updating layouts.
